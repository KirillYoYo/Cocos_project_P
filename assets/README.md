# ✈️ Game Controller & Plane System

### 1. Генерация мира

Все точки кривой заранее создаются методом:

```js
generateCurveVertices(...)
```

Хранятся в массиве:

```js
allPoints
```

Затем массив:

- реверсится;
- используется как стек (`pop()`).

---

### 2. Видимые точки (ключевая оптимизация)

Используются два массива:

- `allPoints` — полный набор (источник);
- `visiblePoints` — только то, что нужно отрисовать.

---

### 3. Добавление точек (справа)

Метод:

```js
fillVisiblePoints()
```

Работает так:

- добавляет точки из `allPoints` → `visiblePoints`;
- делает это **батчами по 10**, между батчами используется:

```js
await requestAnimationFrame()
```

---

### 4. Удаление точек (слева)

Ключевая оптимизация — **без удаления из массива**.  
Вместо этого используется счётчик:

```js
headIndex
```

Логика:

```js
if (точка ушла за экран) {
  headIndex++;
}
```

---

### 5. Периодическая очистка

Раз в 2 секунды вызывается:

```js
cleanupOldPoints()
```

Действия:

- создаётся новый массив;
- копируются только актуальные точки;
- `headIndex` сбрасывается.

---

### 6. Скролл мира

Каждый кадр выполняется:

```js
scrollOffset += speed * dt;
```

👉 Точки визуально сдвигаются за счёт этого смещения (`offset`).

---

### 7. Отрисовка

Метод:

```js
drawPoints()
```

Особенности:

- рисуются только точки в пределах экрана;
- координаты пересчитываются относительно центра;
- используется `Graphics` для рендеринга.

---

## ✈️ Движение самолёта

### 1. Позиция по кривой

Метод:

```js
getCurveYAtCenter()
```

Функция:

- находит сегмент кривой под центром экрана;
- интерполирует `Y` через **Catmull–Rom spline**.

---

### 2. Применение позиции

Установка позиции самолёта:

```js
planeController.setPositionY(y)
```
